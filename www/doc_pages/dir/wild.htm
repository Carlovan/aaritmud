File della directory della wild
Questa cartella contiene tutto ciò che riguarda le informazioni wild in formato immagine e la loro gestione base.
L'immagine di wild non ha dei limiti minimi o massimi precisi, insomma più memoria potete utilizzare e più potete farla grande.
La wild implementata nel codice è a tre dimensioni, quindi i pg potranno, volando o scavando, andare realmente alle coordinate in aria o sottoterra; non è una implementazione AFKMud per chi ne sa.

File altitudes.bmp:
Rappresenta le altitudini della wild, ogni colore è un tipo di altitudine, per i mari, oceani e laghi indica la profondità.

File sectors.bmp:
Rappresenta i tipi di settori della wild, ogni colore è un tipo differente di settore.
In pratica si devono disegnare qui strade e altri tipi di settori per non andare a perdere le informazioni riguardanti le altitudini dell'altra immagine.

File count_wild_colors.py:
È uno script in python per contare quali e quanti colori vi sono nelle immagini di wild.

File create_wild_sectors.py:
Questo script in python invece serve a creare l'immagine dei settori partendo da quella delle altitudini, questo perchè io creo precedentemente quella delle altitudini da un programma di creazione mappe casuali di tipo frattale.
