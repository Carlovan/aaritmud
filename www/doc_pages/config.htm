Il File di Configurazione:

Di seguito sono descritti i singoli campi che riguardano, di default, il file config.ini nella cartella principale di Aarit.

Sezione [CFG]
Impostazioni riguardanti la configurazione e la manutenzione generale del Mud.

port:
Porta web a cui connettersi con il browser, l'80 è quella di default cosicché non v'è bisogno d'inserirla nell'url del browser.

game_name:
Nome del Mud, si possono inserire i tag di stile per colorarlo come si vuole.

smtp_host:
smtp_email:
Serve per inviare mail ai giocatori (tipo per il recupero della password) l'SmtpHost deve essere un indirizzo relativo al vostro gestore di connessione internet, quindi se per esempio utilizzate connessione Libero dovete inserire l'smtp di Libero (altrimenti otterrete un messaggio di errore tipo questo: 550 RCPT TO: Relaying not allowed).

log_accent:
Se impostato a True stamperà alla console i messaggi di log con gli accenti (àèìòù) se invece viene impostato a False gli accenti verranno convertiti in vocali apostrofate (a' e' i' o' u') questo è utile quando alcune console supportano solo i caratteri ASCII.
Bisogna però fare attenzione ad effettuare una ricerca del messaggio di log nel codice per tracciare l'errore, potreste non trovare la stringa esatta poiché effettivamente differente.

development:
Se state sviluppando o testando Aarit meglio tenere questa variabile a True, ciò vi permetterà di riaggiornare le pagine web ogni volta che vengono chiamate con eventuali nuove modifiche, stessa cosa per i moduli dei comandi (in src/commands) che verranno ricaricati ad ogni utilizzo. Inoltre questa configurazione vi permetterà di leggere maggiori informazioni di debug.

min_len_name:
max_len_name:
Lunghezza minima e massima permessa ai nomi di account o di personaggio.
Ricordatevi che eventuali colori inseriti nel nome non inficiano sulla lunghezza totale del nome.

min_len_password:
max_len_password:
Lunghezza minima e massima relativa alla password dell'account, per una maggior sicurezza il minimo dovrebbe essere almeno 7, ma potete sceglierne comunque una minore.

max_aliases:
Valore massimo di alias impostabili per personaggio.
Se come valore inserite lo 0 indicate che non supportate gli aliases.

max_macros:
Valore massimo di macro impostabili per personaggio.
Se come valore inserite lo 0 indicate che non supportate le macro.

max_account_players:
Valore massimo di giocatori creabili per ogni account.
Se come valore inserite lo 0 indicate che non supportate la creazione di nuovi personaggi (può essere utile in casi di emergenza per evitare proliferazione di personaggi nuovi con caratteristiche errate).

min_reset_time:
Tempo minimo di reset per le aree consentito (in minuti reali).

max_reset_time:
Tempo massimo di reset per le aree consentito (in minuti reali).

chars_for_smile:
Numero di caratteri finali controllati in una frase inviata tramite un canale rpg in cui viene cercato uno smile poi convertito in un'espressione, per esempio:
"di ciao :)" -> "dici sorridendo: 'ciao'"
Se non si vuole questa feature basta inserire a 0 tale configurazione.

first_destination:
Destinazione voluta per la prima entrata in gioco da parte dei personaggi.
Per saperne di più su come impostare la destinazione vedere questa pagina.

Sezione [CALENDAR]
Impostazioni riguardanti il calendario RPG del Mud:

dawn_hour:
A che ora rpg è l'aurora.

sunrise_hour:
A che ora rpg è l'alba.

noon_hour:
A che ora rpg è mezzogiorno.

sunset_hour:
A che ora rpg c'è il tramonto.

dusk_hour:
A che ora rpg è il crepuscolo.

midnight_hour:
A che ora rpg è mezzanotte.

seconds_in_minute:
Numero di secondi reali che formano un minuto rpg, il minimo può essere 1 (e in tal caso un giorno RPG nel Mud durerebbe 24 minuti reali).

minutes_in_hour:
Quantità di minuti rpg dentro un'ora rpg.

hours_in_day:
Quantità di ore rpg dentro un giorno rpg.

days_in_month:
Quantità di giorni rpg dentro un mese rpg.
Se si modifica quest'opzione bisogna tenere d'occhio anche l'enumerazione DAYWEEK in src/enums e il metodo get_day_week nella classe Config nel file src/config.py.

months_in_year:
Quantità di mesi rpg dentro un anno rpg.
Se si modifica quest'opzione bisogna anche modificare l'enumerazione MONTH in src/enums.

minute:
hour:
day:
month:
year:
Valori che salvano l'attuale data RPG del mud, il minuto si inizia a contare a 0.
